correct_knife_basic¶

DLNIRSP_Pipeline.IFUCalibration.correct_knife_basic(fitsData, dark_cal, gain_cal, save_dir, threads=1)[source]¶

Do “basic” reductions (dark and flat) on a set of knife-edge scan images

This function just manages the parallelization of the reduction. See basic_helper for the meat of the script. The individual reduced images are saved to disk because the knife datasets are often too large to keep and move around in memory.

Parameters
fitsDataData.FitsData

Object with each knife image to be corrected in a separate data_hdu

dark_calData.FitsData

Dark Calibration object

gain_calData.FitsData

Gain Calibration object

save_dirstr

Directory in which to save the reduced images. Each will have a file name of SAVE_DIR/dg_ORIGINALNAME

threadsint

The number of cpu threads to use. This is an embarassingly parallizable problem, so throw as much as you want at it.
