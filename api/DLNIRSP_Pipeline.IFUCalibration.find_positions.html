find_positions¶

DLNIRSP_Pipeline.IFUCalibration.find_positions(fitsData, thresh=0.5)[source]¶

Find a single axis (x|y) of the IFU FOV position of each spatial pixel in the input data

The input data must represent an ordered sequence of knife-edge (scan mirror) steps across the face of the IFU along a single axis. The positions found by this function will correspond to the direction of knife-edge motion.

The algorithm is as follows:

  1. Assign each scan step a real position in the FOV

  2. Collapse all steps along the wavelength dimension

  3. Filter out positions with no/low signal

  4. For each slit location: 1. Compute the gradient of signal with step 2. Fit the peak in gradient 3. Compute the real position associated with the scan step at the peak gradient

The peak gradient of each spatial position corresponds to the moment the knife-edge passes in front of that particular fiber. Thus, these peaks show the real spatial location of each slit position.

Parameters
fitsDataData.FitsData

Each step of the knife-edge scan should be a separate data_hdu and they should be in order

threshfloat

A number used to reject slit positions with low/no signal

Returns
numpy.ndarray

The real, physical location (along the scan dimension) for each slit position

numpy.ndarray

Array of pixel locations of the slit locations with good data
